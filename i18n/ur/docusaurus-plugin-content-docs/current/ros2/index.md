# ROS 2 (روبوٹ آپریٹنگ سسٹم)

## ROS 2 کا تعارف

روبوٹ آپریٹنگ سسٹم (ROS) روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک ہے۔ یہ اوزاروں، لائبریریوں، اور کنونشنز کا مجموعہ ہے جو مختلف روبوٹک پلیٹ فارمز پر پیچیدہ اور مضبوط روبوٹ رویہ تخلیق کرنے کے عمل کو آسان بناتا ہے۔

ROS 2، اصل ROS کا مکمل دوبارہ ڈیزائن ہے، جو جدید روبوٹکس ایپلیکیشنز کی ضروریات، جیسے ملٹی روبوٹ سسٹمز، ریئل ٹائم کنٹرول، اور کمرشل پروڈکٹس، کو پورا کرنے کے لیے بنایا گیا ہے۔ یہ Data Distribution Service (DDS) اسٹینڈرڈ پر بنایا گیا ہے، جو ایک مضبوط اور قابل توسیع کمیونیکیشن لئیر فراہم کرتا ہے۔

یہ باب ROS 2 کا جامع جائزہ فراہم کرے گا، جس میں اس کی اعلی سطحی آرکیٹیکچر، بنیادی کمیونیکیشن پیٹرنز، اور ڈویلپمنٹ ٹولز شامل ہیں۔

## بنیادی آرکیٹیکچر

ROS 2 بنیادی طور پر پراسیسز (جنہیں "نوڈز" کہا جاتا ہے) کا ایک تقسیم شدہ نظام ہے، جو پیچیدہ کام انجام دینے کے لیے ایک دوسرے کے ساتھ بات چیت کرتے ہیں۔

### DDS (ڈیٹا ڈسٹریبیوشن سروس)

اصل ROS کے برعکس، جس میں کسٹم کمیونیکیشن پروٹوکول استعمال ہوتا تھا، ROS 2 DDS کے اوپر بنایا گیا ہے۔ DDS ایک صنعتی معیار کا مڈل ویئر ہے جو ریئل ٹائم اور ایمبیڈڈ سسٹمز کے لیے استعمال ہوتا ہے۔ یہ پبلش-سبسکرائب کمیونیکیشن ماڈل فراہم کرتا ہے جو غیر مرکزی، قابل توسیع، اور انتہائی قابل اعتماد ہے۔

DDS استعمال کرنے سے ROS 2 کو کئی خصوصیات حاصل ہوتی ہیں، جیسے:

- **غیر مرکزی دریافت (Decentralized Discovery)**: نوڈز خود بخود نیٹ ورک پر ایک دوسرے کو دریافت کر سکتے ہیں بغیر کسی مرکزی ماسٹر کے۔
- **سروس کی معیار (QoS)**: ROS 2 مختلف QoS پالیسیز فراہم کرتا ہے تاکہ ڈویلپرز نوڈز کے درمیان کمیونیکیشن کو مخصوص حالات کے لیے بہتر بنا سکیں (جیسے اعتبار، دیرپا پن، تاخیر)۔
- **انٹرآپریبلٹی**: مختلف DDS ایمپلیمینٹیشنز ایک دوسرے کے ساتھ کام کر سکتی ہیں۔ ROS 2 کئی DDS وینڈرز کو سپورٹ کرتا ہے، جیسے eProsima Fast DDS (ڈیفالٹ)، RTI Connext، اور Eclipse Cyclone DDS۔

### نوڈز

نوڈ ROS 2 سسٹم کی بنیادی اکائی ہے۔ یہ ایک پراسیس ہے جو مخصوص کام انجام دیتا ہے، جیسے:

- موٹر کنٹرول کرنا
- سینسر سے ڈیٹا پڑھنا
- روبوٹ کے لیے راستہ منصوبہ بندی کرنا
- ڈیٹا کو ویژولائز کرنا

نوڈز عام طور پر C++ یا Python میں ROS 2 کلائنٹ لائبریریز (`rclcpp` اور `rclpy`) کے ساتھ لکھے جاتے ہیں اور انہیں ملا کر پیچیدہ روبوٹ رویے تیار کیے جا سکتے ہیں۔

### ROS گراف

ROS گراف ROS 2 نوڈز اور ان کے کنیکشنز کا نیٹ ورک ہے۔ یہ اس بات کی تصوراتی نمائندگی ہے کہ سسٹم کے مختلف حصے کس طرح ایک دوسرے کے ساتھ بات چیت کر رہے ہیں۔ `rqt_graph` جیسے ٹولز ROS گراف کو دیکھنے اور نوڈز کے درمیان کمیونیکیشن ڈیبگ کرنے کے لیے استعمال ہوتے ہیں۔

## کمیونیکیشن پیٹرنز

ROS 2 نوڈز کے درمیان ڈیٹا کے تبادلے کے لیے کئی کمیونیکیشن پیٹرنز فراہم کرتا ہے۔

### ٹاپکس (Topics)

ٹاپکس ROS 2 میں سب سے عام کمیونیکیشن پیٹرن ہیں۔ یہ ایک غیر ہم وقت، ایک سے زیادہ چینل پر مبنی کمیونیکیشن فراہم کرتے ہیں، جو پبلش-سبسکرائب ماڈل پر کام کرتا ہے۔

- ایک نوڈ **پیغام شائع** (publish) کر سکتا ہے۔
- کوئی بھی دیگر نوڈز اس ٹاپک کو **سبسکرائب** کر کے پیغامات وصول کر سکتے ہیں۔

ٹاپکس مسلسل ڈیٹا اسٹریم کے لیے مثالی ہیں، جیسے سینسر ڈیٹا (کیمرہ امیجز، LiDAR اسکینز) یا روبوٹ کی حالت کی معلومات (جیسے جوائنٹ پوزیشنز، اوڈومیٹری)۔

### سروسز (Services)

سروسز ہم وقت، ایک سے ایک کمیونیکیشن پیٹرن فراہم کرتی ہیں، جو ریکوئسٹ-ریسپانس ماڈل پر مبنی ہے۔

- ایک **کلائنٹ** نوڈ **ریکوئسٹ** بھیجتا ہے۔
- سرور نوڈ اس ریکوئسٹ کو پروسیس کر کے **جواب** دیتا ہے۔

سروسز چھوٹے، ٹرانزیکشنل تعاملات کے لیے استعمال ہوتی ہیں، جیسے:

- روبوٹ کی موجودہ حالت جانچنا
- مخصوص ایکشن کو ٹرگر کرنا (مثلاً تصویر لینا)
- کوئی حساب کتاب کرانا

### ایکشنز (Actions)

ایکشنز طویل مدتی، غیر ہم وقت کاموں کے لیے استعمال ہوتی ہیں جو عمل کے دوران فیڈبیک فراہم کرتی ہیں۔

- ایک **ایکشن کلائنٹ** **گول** بھیجتا ہے ایک **ایکشن سرور** کو۔
- ایکشن سرور گول پر عمل شروع کرتا ہے اور کلائنٹ کو وقفے وقفے سے **فیڈبیک** دیتا ہے۔
- جب کام مکمل ہو جاتا ہے، سرور حتمی **نتیجہ** کلائنٹ کو بھیجتا ہے۔

ایکشنز ایسے کاموں کے لیے مثالی ہیں جیسے:

- کسی مخصوص مقام تک جانا
- پیچیدہ مینپولیشن ٹاسک انجام دینا
- پہلے سے طے شدہ ٹریکٹری کو فالو کرنا

کلائنٹ کسی بھی وقت گول کو منسوخ بھی کر سکتا ہے۔

## Python انٹیگریشن (rclpy)

`rclpy` ROS 2 کی آفیشل Python کلائنٹ لائبریری ہے۔ یہ Python میں ROS 2 کے تمام بنیادی تصورات کے لیے انٹرفیس فراہم کرتی ہے، جس سے ڈویلپرز نوڈز، پبلشرز، سبسکرائبرز، سروسز، اور ایکشنز Python میں لکھ سکتے ہیں۔

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class HelloWorldPublisher(Node):

    def __init__(self):
        super().__init__('hello_world_publisher')
        self.publisher_ = self.create_publisher(String, 'hello_world', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello, ROS 2!'
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    hello_world_publisher = HelloWorldPublisher()
    rclpy.spin(hello_world_publisher)
    hello_world_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
